## What the header profile dropdown menu is
The menu that pops out of the profile button in the main header is implemented inside `components/header.tsx`. It is a small custom popover (not Radix `DropdownMenu`) that is rendered conditionally when `profileMenuOpen` is true.

It provides quick navigation to the auth portals (buyer/vendor/services) and a small internal toggle between two display modes:
- `profileMenuMode = "login"` shows "Login as ..." links and a "Join MyGarage" button.
- `profileMenuMode = "join"` shows "Join as ..." links and a "My Account" button that switches back to login mode.

There is no sign-out action in this header dropdown. Sign-out is implemented inside the portal sidebars (buyer/vendor/services/admin).

## State and refs (must match)
In `Header()`:
- `const [profileMenuOpen, setProfileMenuOpen] = useState(false)`
- `const [profileMenuMode, setProfileMenuMode] = useState<'login' | 'join'>('login')`
- `const profileCloseTimerRef = useRef<number | null>(null)`
- `const mobileProfileMenuRef = useRef<HTMLDivElement | null>(null)`
- `const desktopProfileMenuRef = useRef<HTMLDivElement | null>(null)`

The ref wrappers are used for click-outside detection on both mobile and desktop header layouts.

## Open/close behavior (exact)
### 1) Click to toggle
Trigger button calls:
- `toggleProfileMenu()` which does:
  - `setProfileMenuOpen((prev) => !prev)`

### 2) Hover to open/close (with delay)
The wrapper `<div className="relative">` around the profile button has:
- `onMouseEnter={handleProfileHoverOpen}`
- `onMouseLeave={scheduleProfileHoverClose}`

Functions:
- `handleProfileHoverOpen()`:
  - clears any pending close timer
  - sets `profileMenuOpen = true`

- `scheduleProfileHoverClose()`:
  - clears any pending close timer
  - schedules a close after 150ms:
    - `setProfileMenuOpen(false)`
    - `setProfileMenuMode('login')` (always resets mode when it closes)

This means the menu:
- opens immediately on hover
- closes shortly after pointer leaves the wrapper
- always returns to the "login" view the next time it opens

### 3) Click outside closes the menu
`Header` registers a global mousedown handler:
- `document.addEventListener('mousedown', onPointerDown)`

`onPointerDown` checks whether the click target is inside either wrapper:
- `mobileProfileMenuRef.current?.contains(target)`
- `desktopProfileMenuRef.current?.contains(target)`

If the click is outside both:
- `setProfileMenuOpen(false)`

Note: click-outside does not explicitly reset `profileMenuMode`; the hover-close path does reset mode, but click-outside only closes.

## DOM structure and Tailwind classes (must match)
The profile menu appears in both mobile and desktop header sections, with essentially the same structure and classes.

### Wrapper
- `<div ref={mobileProfileMenuRef or desktopProfileMenuRef} className="relative" ...hover handlers...>`

### Trigger button
Button (same on mobile + desktop):
- `type="button"`
- `aria-label="Open profile menu"`
- `aria-expanded={profileMenuOpen}`
- `onClick={toggleProfileMenu}`
- classes:
  - `inline-flex items-center gap-1 rounded-lg border border-border bg-background px-3 py-2 text-sm text-foreground hover:bg-accent hover:text-accent-foreground transition`
- icons inside:
  - `<UserCircle2 className="h-5 w-5" />`
  - `<ChevronDown className="h-4 w-4" />`

### Popover container (only when open)
Rendered only when `profileMenuOpen` is true:
- `<div className="absolute right-0 mt-2 w-56 rounded-lg border border-border bg-popover p-2 text-popover-foreground shadow-lg z-50">`

Inside, the menu content is conditional on `profileMenuMode`.

## Menu content: "login" mode
When `profileMenuMode === 'login'` the popover renders:

1) Buyer login link:
- `<Link href="/auth?role=buyer&next=/buyer" className="block rounded-md px-3 py-2 text-sm hover:bg-accent hover:text-accent-foreground transition">Login as Buyer</Link>`

2) Vendor login link:
- `<Link href="/auth?role=vendor&next=/vendor" className="block rounded-md px-3 py-2 text-sm hover:bg-accent hover:text-accent-foreground transition">Login as Vendor</Link>`

3) Service Provider login link:
- `<Link href="/auth?role=services&next=/services/orders" className="block rounded-md px-3 py-2 text-sm hover:bg-accent hover:text-accent-foreground transition">Login as Service Provider</Link>`

4) Divider:
- `<div className="my-1 h-px bg-border" />`

5) Mode switch button:
- `<button type="button" onClick={() => setProfileMenuMode('join')}>Join MyGarage</button>`
- classes:
  - `block w-full rounded-md px-3 py-2 text-left text-sm font-medium text-primary hover:bg-accent transition`

## Menu content: "join" mode
When `profileMenuMode !== 'login'` (i.e. `"join"`) the popover renders:

1) Buyer join link:
- `<Link href="/auth?role=buyer&next=/buyer" className="block rounded-md px-3 py-2 text-sm hover:bg-accent hover:text-accent-foreground transition">Join as Buyer</Link>`

2) Vendor join link:
- `<Link href="/auth?role=vendor&next=/vendor" className="block rounded-md px-3 py-2 text-sm hover:bg-accent hover:text-accent-foreground transition">Join as Vendor</Link>`

3) Service Provider join link:
- `<Link href="/auth?role=services&next=/services/orders" className="block rounded-md px-3 py-2 text-sm hover:bg-accent hover:text-accent-foreground transition">Join as Service Provider</Link>`

4) Divider:
- `<div className="my-1 h-px bg-border" />`

5) Mode switch button:
- `<button type="button" onClick={() => setProfileMenuMode('login')}>My Account</button>`
- classes:
  - `block w-full rounded-md px-3 py-2 text-left text-sm font-medium text-primary hover:bg-accent transition`

## Accessibility notes (what is present)
- The trigger button uses `aria-expanded`.
- The popover does not add `role="menu"` or focus trapping; it relies on click-outside + hover leave to close.
